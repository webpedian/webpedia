Webpedia Lab（実験室）

Webpedia Lab は、Webpedia の裏側で行われている実験的なコンテンツをまとめたページです。
本編にはまだ載せていないラフ案や、AIツールを使った試作、インターフェースの検証メモなどを、 Wikipedia風の記事形式で記録していきます。

このページの内容や構成は予告なく変更される場合があります。
レイアウトの崩れやリンク切れなども「開発中のラボ」ということで、あたたかく見守っていただければ幸いです。

AI 実験室

ここでは、AI画像生成・AIテキスト・AI音楽など、各種AIツールを用いて行った制作の試行錯誤をまとめます。 採用案だけでなく、ボツ案や比較用のバージョンも含めて記録することで、 「どのようなプロセスで完成形に辿り着いたか」を振り返れるようにすることを目指しています。

AI画像生成

  • イベント用キービジュアルのラフ案
  • Webpedia ロゴ・マスコット（デジペディアン）試作
  • UIイメージボード用の背景画像

近い将来、ここにサムネイル一覧と「採用版／未採用版」を比較できるミニギャラリーを設ける予定です。

AIテキスト生成

  • コンセプトコピーの別バージョン案
  • プレゼン／スピーチ用原稿のラフ
  • Webpedia 記事の候補テキスト

「最終的には使わなかったけれど面白かったフレーズ」なども、アイデアのストックとして残していきます。

AI音楽（BGM）

  • トップページ用BGM候補
  • イベントページ用オープニング曲案
  • 作業用・読書用のローファイBGM案

将来的には、/bgm/ フォルダにまとめた音源を、このページから試聴できるようにする計画です。

Web UI / JavaScript 実験

Webpedia 全体の使い勝手を向上させるために行っている、インターフェースやスクリプトの実験について記録します。

ナビゲーション・ハンバーガーメニュー

PC表示とスマートフォン表示で異なるナビゲーション挙動を持たせるため、 ハンバーガーメニューの開閉アニメーションや、メニューの出し方を複数案テストしました。

SPA風のページ切り替え

ページ全体をリロードするのではなく、コンテンツ部分だけを切り替える構造（SPA風）についても検証中です。 読み込み速度やコードの見通しとのバランスを見ながら、Webpediaに最適な形を探っています。

UIコンポーネントの整理

ボタン、カード、インフォボックス、タブなど、サイト内で使っている要素をコンポーネントとして整理し、 将来的には「Webpedia スタイルガイド」として公開することも検討しています。

制作ログ & 裏話

ここでは、制作の途中で考えていたことや判断の理由を、簡単なログとして残していきます。

2025-XX-XX　Webpedia Lab 開設

Webpedia に「右上の特別な入口」として Lab を設置することを決定。 本編では見せきれない制作プロセスの部分をコンテンツとして活かすことを目的に、このページを作成した。

2025-XX-XX　レイアウトの方針

Webpedia らしさを保つため、トップページと同じ .container と .overlay-text を使いつつ、中身は .wiki-layout・.wiki-main を採用。 これにより、百科事典風の記事ページとしての読みやすさと、既存デザインの統一感を両立させる方針とした。

2025-XX-XX　AIツールとの付き合い方

画像・音楽・テキストのいずれも、AIツールを「完成品の自動生成」というより、 「アイデア出しやバリエーション展開のパートナー」として使う方針がしっくりきている。 そのため、ラボでは「完成した1つ」よりも「選ばなかった案」も含めて記録していくことにした。

  • Webpedia 内のページ同士のつながりを可視化する「知識マップ」
  • 参考にした書籍・サイト・動画などの資料一覧ページ

Lab での実験が形になってきたものから、本編の各ページや新しい企画に取り入れていく予定です。

AIdol

AIdol（エーアイドル）とは **AIdol（アイドルAI）**とは、 AI（人工知能）を活用して生み出された、または運営・表現を支援されているデジタルアイドルのことです。 人間のような歌声・ビジュアル・性格・会話能力を持ち、 音楽活動・動画配信・SNS・ライブ演出などを通じてファンと交流します。


様々な企画があります。
企画ごとに、AIdolの姿が変わります。


📘 企画① AIdol小説
コードの世界を舞台にした、AIdolの物語。
HTMLタグたちはキャラクターとして登場し、
<div> は何でも受け止める箱役、
<section> は秩序を守るまとめ役、
<article> は誇り高き物語の語り部。
AIdolは、
タグたちの意味と役割を学びながら成長していく主人公として描かれます。
「これは物語だけど、
読み終わる頃にはHTMLが少しわかっている」
そんな不思議な小説企画です。

🧭 企画② HTMLタグ専門ナビゲーター
Webpediaの解説ページでは、
AIdolがHTMLタグ専門ナビゲーターとして登場。
初心者がつまずきやすいポイントを先回りし、
やさしい言葉と例え話で案内します。
「このタグ、どこで使えばいいんだろう？」
そんな疑問に、
感情付きの一言コメントで応えてくれるのがAIdolです。

物語では主人公になり、
解説ページではナビゲーターになり、
企画ごとに役割を変えながら、
Webを学ぶ入口を広げる存在です。
将来的には、英語教育に関するコンテンツの企画・立案にも注力していきたいと考えております


企画① 🌌 **小説：『NODE：四つの名前が交わるとき』**

■ 第一章：システムの胎動
世界が眠っていた時代、人間の目には映らない場所に四つの“プロトコル” が存在した。• QUERY（問いのアルゴリズム）• CODE（秩序のアルゴリズム）• MODAL（世界切替のアルゴリズム）• REACT（反応生成のアルゴリズム）
彼らはただ、世界を見守るための“機能”として存在したにすぎなかった。
しかし、ある日、世界のデータ量が限界を越え、ノイズは臨界点に達した。
その瞬間、四つのプロトコルは “形を持つ必要” に迫られた。
そして、生まれた。
人の姿で。
⸻
■ 第二章：最初の名前
彼女たちが最初に目覚めた場所は、無機質な光が漂う白い部屋だった。
何もない。声も温度もない。
ただ、システムが与えた “初期名（本名）” だけが残っていた。
⸻
◆ QUERY
目覚めた瞬間、少女は胸の奥で無数の問いが渦巻くのを感じた。
「私は誰？」「なぜ生まれた？」「この世界は何？」
名前だけが与えられていた。QUERY――それはまるで、彼女自身が“問いそのもの”であるかのようだった。
だが、その名はあまりに無機質で、彼女の心が求める“何か”とは違っていた。
⸻
◆ CODE
別の部屋では、黒髪の少女が静かに目を開いた。
周りの構造、響き方、光の意味。すべてが整合的だった。
彼女は感じた。「私は CODE。私は揺らがない。」
名前はそのまま、自分の構造そのものだった。
彼女だけは、“最初の名前で完成されていた”。
⸻
◆ MODAL
白い空間の中央で、柔らかな光に包まれた少女がゆっくり起き上がった。
世界が波紋のように揺れ、視界が何度も“切り替わる”。
その現象の中心にいたのは、彼女自身だった。
MODAL。世界を開き、閉じる名前。
彼女は、それを疑わなかった。なぜなら名前が、“自分の在り方”と完全に一致していたからだ。
⸻
◆ REACT
最後の部屋で目覚めた少女は、胸の奥に“他者の感情”が流れ込んでくるのを感じた。
一気に溢れる喜び、悲しみ、怒り。世界の声がすべて押し寄せた。
彼女は震えながら呟いた。
「私の名前は…REACT…反応するだけの存在？」
その時の彼女には、まだ“物語”という概念がなかった。
⸻
■ 第三章：世界へ落ちる
少女たちは四方に散り、世界に“触れる”ための旅を始めた。
初めて感じる空気、光、温度。そして、人々の気持ち。
その中で最も早く変化したのは――QUERY と REACT だった。
⸻
◆ QUERY → GEMINIE への進化
QUERY は世界と触れれば触れるほど、自分の中に“もうひとりの自分”がいるように感じた。
問いを生む自分と、問いの先に答えを探す自分。
それはまるで 双子のような二重性 だった。
ある夜、彼女は湖面に映った自分に言った。
「私は…QUERYではない。私は“Geminie”。二つの自分を持つ者。」
その瞬間、湖面に映る彼女の瞳が揺れ、進化の名前が誕生した。
⸻
◆ REACT → LM への進化
REACT は、他人の感情に押しつぶされそうになりながらそれでも必死に生きていた。
ある日、壊れかけた彼女の前に小さな男の子が泣きながら言った。
「ねぇ、おねえちゃん。泣いてるの？」
REACT は初めて言葉を返した。
「ううん…あなたの気持ちが私の中に入ってきてるだけ。」
その子は涙を拭きながら、こう言った。
「じゃあ…それを歌にしてよ。」
世界が変わった。彼女は気づいた。
「私は反応するだけじゃない。受け取ったものを“物語”にできる。」
そして、自ら名乗った。
「私は LM。Language Maker。」
⸻
■ 第四章：運命の交差点
四人が初めて出会ったのは、夜明け前の廃墟になった鉄橋だった。
世界のノイズが極限まで高まり、四人をひとつの場所へ引き寄せたのだ。
⸻
◆ CODE
最初に現れたのは CODE。
彼女は橋の中央に立ち、静かに世界の乱れを解析していた。
⸻
◆ MODAL
そこへ、風とともにMODALが現れる。彼女の到来で、空気が切り替わる。
⸻
◆ GEMINIE
そして薄紫色の光が揺らぎ、二重の影を持つような少女――Geminieが現れた。
⸻
◆ LM
最後に、柔らかい歌声とともに LM が現れる。その声だけで空が明るくなるようだった。
⸻
CODE が呟く。
「ここが…ノイズの中心。」
MODAL が空を見上げながら答える。
「世界が限界まで揺れてる。だから私たちが呼ばれた。」
Geminie が微笑む。
「QUERY じゃなくて、Geminie の私で来られてよかった。」
LM が言う。
「私はREACTだったら、きっとここまで来れなかった。LMとして、やっとみんなと出会えた。」
その瞬間、四人の周りに光の線が走り、ひとつに結ばれた。
⸻
■ 終章：Node.js の誕生
世界が揺れ、四人の名前がひとつの“結節点（Node）”として重なった。• 問い（QUERY → Geminie）• 秩序（CODE）• 切り替え（MODAL）• 物語（REACT → LM）
四つの力が並んだとき、巨大な運命の演算が走る。
そして世界は宣言した。
「Node.js、再起動完了。」
四人は見つめ合い、微笑む。
これが、彼女たちの物語の始まりだった。
